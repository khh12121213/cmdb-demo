# 银行 CI/CD-CMDB 系统 — 部署文档

> 版本：v0.1.0 | 适用环境：银行内网 RHEL/CentOS 7+/Ubuntu 20.04+

---

## 1. 环境要求

| 组件 | 最低版本 | 说明 |
|------|---------|------|
| Python | 3.12+ | 异步协程依赖，3.10 已于 2026 年 EOL |
| MySQL | 8.0+ | 主数据存储，utf8mb4 字符集 |
| Redis | 5.0+ | 分布式锁 + 变量缓存 |
| Node.js | 18+ | 前端构建，支持 ESM |
| 操作系统 | Linux x64 | 2 核 4G 起 |

## 2. 基础设施准备

### 2.1 MySQL

```sql
-- 创建数据库
CREATE DATABASE bank_cicd_cmdb
  DEFAULT CHARACTER SET utf8mb4
  DEFAULT COLLATE utf8mb4_unicode_ci;

-- 创建专用账号
CREATE USER 'cicd_admin'@'%' IDENTIFIED BY '你的密码';
GRANT ALL PRIVILEGES ON bank_cicd_cmdb.* TO 'cicd_admin'@'%';
FLUSH PRIVILEGES;
```

建表由应用启动自动完成（SQLAlchemy `create_all`），无需手动执行。

> **生产建议**：手动执行建表脚本后关闭自动建表，避免误改线上表结构。

### 2.2 Redis

```bash
# 安装 Redis 5.0+（已有则可跳过）
yum install -y redis  # or apt install redis-server

# 确认版本
redis-cli --version
# 预期: redis-cli 5.0.x 或更高

# 启动
systemctl enable redis && systemctl start redis
```

无需特殊配置，默认 6379 端口、无密码即可对接。生产环境建议设置密码。

## 3. 后端部署

### 3.1 获取代码

```bash
# 将 cmdb-demo 目录整体拷贝到目标服务器
scp -r cmdb-demo/ user@target-server:/data/cmdb/
cd /data/cmdb/cmdb-demo
```

### 3.2 创建虚拟环境

```bash
python3.12 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### 3.3 修改配置文件

编辑 `.env`：

```ini
# 监听地址和端口
SERVER_HOST=0.0.0.0
SERVER_PORT=8000
ENV_MODE=prod

# MySQL
DB_HOST=你的MySQL地址
DB_PORT=3306
DB_USER=cicd_admin
DB_PWD=你的密码
DB_NAME=bank_cicd_cmdb

# Redis
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
REDIS_PASSWORD=
REDIS_DB=0

# AES 加密密钥（生产环境务必更换为32字节随机串，对接 KMS 轮换）
AES_SECRET_KEY=你的32字节密钥--change-me!!
```

### 3.4 启动服务

```bash
source venv/bin/activate

# 前台启动（调试）
uvicorn app.main:app --host 0.0.0.0 --port 8000

# 生产启动（systemd）
# 见 5.1 节 systemd 配置
```

### 3.5 验证

```bash
# 健康检查
curl http://127.0.0.1:8000/api/health
# 预期: {"status":"ok"}

# 数据库连通检查
curl http://127.0.0.1:8000/api/ready
# 预期: {"status":"ok","db":"connected"}

# API 文档
curl http://127.0.0.1:8000/docs
# 浏览器访问即可看到 Swagger 交互文档
```

## 4. 前端部署

### 4.1 安装依赖

```bash
cd /data/cmdb/cmdb-demo/frontend
npm install --registry=https://registry.npmmirror.com
```

### 4.2 构建生产包

```bash
npx vite build
# 产物在 dist/ 目录
```

### 4.3 Nginx 配置（推荐）

```nginx
server {
    listen 80;
    server_name cmdb.infra.bank.com;  # 替换为实际域名

    # 前端静态文件
    root /data/cmdb/cmdb-demo/frontend/dist;
    index index.html;
    try_files $uri $uri/ /index.html;

    # API 代理到后端
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

```bash
nginx -t && nginx -s reload
```

### 4.4 开发模式（本地/测试）

```bash
cd frontend
npx vite --host 0.0.0.0 --port 3000
# 浏览器访问 http://服务器IP:3000
```

## 5. 生产加固

### 5.1 systemd 服务单元

创建 `/etc/systemd/system/cmdb-demo.service`：

```ini
[Unit]
Description=银行 CI/CD CMDB 后端服务
After=network.target mysql.service redis.service

[Service]
Type=simple
User=appdeploy
WorkingDirectory=/data/cmdb/cmdb-demo
Environment=PATH=/data/cmdb/cmdb-demo/venv/bin:/usr/local/bin:/usr/bin
ExecStart=/data/cmdb/cmdb-demo/venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 4
Restart=always
RestartSec=5
LimitNOFILE=65536

[Install]
WantedBy=multi-user.target
```

```bash
systemctl daemon-reload
systemctl enable cmdb-demo
systemctl start cmdb-demo
```

### 5.2 AES 密钥安全

- 禁止将 `.env` 文件提交到 Git
- 生产密钥对接内网 KMS 配置中心，支持定期轮换
- 当前 demo 密钥仅用于测试，**正式上线前必须更换**

### 5.3 日志

本服务使用 `loguru`（控制台输出）。生产建议配合 journald：

```bash
journalctl -u cmdb-demo -f
```

如需文件日志，在启动命令中添加环境变量或修改 `app/config.py`。

### 5.4 防火墙

```bash
# 仅开放必要端口
firewall-cmd --add-service=http  --permanent   # Nginx 80
firewall-cmd --add-port=8000/tcp --permanent   # 后端（仅限内网）
firewall-cmd --reload
```

### 5.5 备份建议

- MySQL：日全量备份 + binlog 增量
- 配置文件：`.env` 纳入配置管理（如 Ansible）
- 审计日志表 `sys_audit_log`：永久保留，不可物理删除

## 7. 快速排障

| 现象 | 检查 |
|------|------|
| 后端起不来 | `ss -tlnp \| grep 8000` 确认端口未被占用 |
| 数据库连不上 | `mysql -u cicd_admin -h 127.0.0.1 -p` 测试连通 |
| Redis 连不上 | `redis-cli ping` 应返回 PONG |
| 前端白屏 | 浏览器 F12 → Console 查看 JS 报错 |
| 接口 404 | 确认路径带 `/api/v1/` 前缀 |
| CORS 报错 | 后端已配置 `allow_origins=["*"]`，检查 Nginx 是否拦截 |
